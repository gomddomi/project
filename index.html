킥오프 논의 목적

이번 회의는 완성본을 확정하는 자리가 아니라, 9월 말 MVP 오픈 범위와 각 팀의 검토 과제를 정렬하는 자리입니다.

검토 요청사항

  • MVP 범위가 9월 말 오픈 기준으로 적절한가?
  • 7종 콘텐츠 상품 구조가 고객에게 이해되는가?
  • 무료체험, API Key, 구독 전환 흐름이 자연스러운가?
  • 마이페이지에서 사용량, 청구, 계정관리 흐름이 충분한가?
  • 디자인 관점에서 누락된 상태/오류/빈 화면은 없는가?
  • 개발/운영 관점에서 정책, API, CMS 리스크는 없는가?

회의 진행 순서

  1. 프로젝트 배경과 고도화 필요성 공유
  2. 기획안 핵심 방향 설명
  3. 프로토타입 주요 흐름 시연
  4. MVP 포함/제외 범위 논의
  5. 디자인, 개발, 콘텐츠, 운영팀 검토사항 확인
  6. 결정사항과 다음 액션아이템 정리

오늘 결정하고 싶은 항목

  • 9월 말 MVP 오픈 범위
  • 카드뉴스 제외 또는 보조 포맷 처리 여부
  • 7종 콘텐츠 상품의 최소 노출 범위
  • 무료체험/API Key/구독 전환 포함 여부
  • 위뉴온 CMS 최소 개발 범위
  • 팀별 후속 검토 담당자와 기한
